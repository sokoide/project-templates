# C++ バージョン別主要機能・進化の歩み

本ドキュメントは、C++ の標準規格 C++98 から最新の C++23 までの歴史における、
主要な機能追加および改善点をまとめたものです。

## 概要

C++ は Bjarne Stroustrup によって 1979 年に「C with Classes」として誕生し、
1998 年に最初の国際標準規格（ISO/IEC 14882）が制定されました。
以降、約 3 年周期で規格改訂が行われています。
C++11 で言語仕様の大幅な近代化が行われたことが最大の転換点であり、
以降の規格（C++14/17/20/23）は継続的な進化を続けています。

---

## バージョン別主要ハイライト

| バージョン | 主な機能追加・改善点 |
| :--- | :--- |
| **C++98** | 最初の国際標準規格。STL、例外、名前空間、テンプレート、RTTI |
| **C++03** | バグ修正中心。値初期化の明確化など |
| **C++11** | `auto`、ラムダ、スマートポインタ、`enum class`、`constexpr`、ムーブセマンティクス |
| **C++14** | 汎用ラムダキャプチャ、`constexpr` の緩和、`std::make_unique` |
| **C++17** | 構造化束縛、`optional`/`variant`/`string_view`、`if constexpr`、CTAD |
| **C++20** | Concepts、三方比較（`<=>`）、Ranges、Coroutines、Modules、`std::span` |
| **C++23** | `std::expected`、多次元 `operator[]`、`if consteval`、Deducing `this` |
| **C++26** | （策定中）`std::linalg`、静的リフレクション、契約プログラミング |

---

## 歴史的転換点

### 1. C++11 — 言語の大規模近代化

C++ の歴史において最大の転換点。C++98 から 13 年の歳月を経て、
ムーブセマンティクス、ラムダ式、スマートポインタなど、
現代的なプログラミングパラダイムが一気に導入されました。
「モダン C++」という呼称は C++11 以降を指します。

### 2. C++20 — テンプレートと並行処理の進化

Concepts によりテンプレートのエラーメッセージが劇的に改善され、
Ranges により要素操作がパイプライン形式で書けるようになりました。
Coroutines と `std::jthread` により、
並行・非同期プログラミングの表現力が大幅に向上しました。

### 3. C++23 — エラーハンドリングと表現力の向上

`std::expected` により例外を使わないエラーハンドリングが標準化され、
`std::print` により `printf`/`iostream` に代わる型安全な出力が導入されました。
Deducing `this` によりメンバ関数のオーバーロードが簡潔に書けるようになりました。

### 4. 標準ライブラリの拡充

各バージョンで標準ライブラリが着実に拡充されており、
Boost に依存していた機能が次々と標準化されています
（`filesystem`、`optional`、`variant`、`format`、`flat_map` など）。

## バージョン間の関係

```text
C++98 → C++03 → C++11 → C++14 → C++17 → C++20 → C++23 → C++26(策定中)
                  ↑
              大規模近代化
```

- **C++98/03**: 古典的 C++（「C with Classes」の延長）
- **C++11/14**: モダン C++（ムーブ、ラムダ、スマートポインタ）
- **C++17/20**: ポストモダン C++（Concepts、Ranges、Coroutines）
- **C++23/26**: コンポーザブル C++（`expected`、モナド操作、リフレクション）
