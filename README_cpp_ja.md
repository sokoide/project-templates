# C++ バージョン別主要機能・進化の歩み

本ドキュメントは、C++ の標準規格 C++98 から最新の C++23 までの歴史における、
主要な機能追加および改善点をまとめたものです。

## 概要

C++ は Bjarne Stroustrup によって 1979 年に「C with Classes」として誕生し、
1998 年に最初の国際標準規格（ISO/IEC 14882）が制定されました。
以降、約 3 年周期で規格改訂が行われています。
C++11 で言語仕様の大幅な近代化が行われたことが最大の転換点であり、
以降の規格（C++14/17/20/23）は継続的な進化を続けています。

---

## バージョン別主要ハイライト

### 古典的 C++（C++98/03）

| バージョン | 主な機能追加・改善点 |
| :--- | :--- |
| **C++98** | 最初の国際標準規格。STL、例外、名前空間、テンプレート、RTTI |
| **C++03** | バグ修正中心。値初期化の明確化など |

### 言語リセット — C++11（2011）

事実上の「新言語」。C++98 とは別言語と言えるほどの大変更。

| カテゴリ | 機能 |
| :--- | :--- |
| 型推論 | `auto`、decltype |
| 制御構文 | range-based for、lambda 式 |
| リソース管理 | move semantics、`unique_ptr`/`shared_ptr` |
| 型安全性 | `nullptr`、`enum class`、`constexpr` |
| テンプレート | 可変長テンプレート（variadic templates） |
| 並行処理 | `std::thread`、`std::mutex`、`std::atomic` |
| その他 | 初期化子リスト、delegating constructor |

### 安定化 — C++14（2014）

C++11 を「まともに使える状態」にしたマイナーアップデート。

| バージョン | 主な機能追加・改善点 |
| :--- | :--- |
| **C++14** | generic lambda、`auto` 戻り値推論、`constexpr` 強化、`std::make_unique` |

### 実務完成度アップ — C++17（2017）

テンプレート地獄をかなり軽減し、実務での完成度が一気に上がりました。

| カテゴリ | 機能 |
| :--- | :--- |
| 言語 | 構造化束縛、**`if constexpr`**、fold 式、CTAD |
| ライブラリ | `optional`、`variant`、`any`、`string_view`、`filesystem` |

### 設計思想の進化 — C++20（2020）

C++11 に次ぐ第二の大革命。「モダン C++ 完成形」に最も近い規格です。

| カテゴリ | 機能 |
| :--- | :--- |
| 言語 | **Concepts**（テンプレート制約）、**Ranges**、Coroutines、Modules |
| 言語 | 三方比較（`<=>`）、`consteval`、指定付き初期化子 |
| ライブラリ | `std::span`、`std::format`、Ranges ライブラリ |

### 現実適用 — C++23（2023）

革命ではなく実用化。C++20 を「現場で使える形」に整備しました。

| カテゴリ | 機能 |
| :--- | :--- |
| 言語 | Deducing `this`、`if consteval`、多次元 `operator[]` |
| ライブラリ | **`std::expected`**、`std::print`、`std::mdspan`、Ranges 強化 |

### 策定中 — C++26

| バージョン | 主な機能追加・改善点 |
| :--- | :--- |
| **C++26** | （策定中）`std::linalg`、静的リフレクション、契約プログラミング |

## 実務での選び方：どのバージョンをターゲットにすべきか？

プロジェクトの要件や環境に応じて、どの C++ バージョンを採用すべきかの指針です。

| ターゲット | 推奨バージョン | SWE が喜ぶポイント |
| :--- | :--- | :--- |
| **レガシー保守** | **C++11** | スマートポインタと `auto` だけで、メモリ管理のバグが激減します。 |
| **標準的な新規開発** | **C++17** | `optional`/`string_view` で安全性と速度を両立 |
| **高度なライブラリ設計** | **C++20** | Concepts により、テンプレートのコンパイルエラーが「人間に読める」ようになります。 |
| **最新・安全性重視** | **C++23** | `expected` により例外に頼らないクリーンなエラー処理が可能になります。 |

## 歴史的転換点：SWE の視点から

### 1. C++11 — 言語の大規模近代化

C++ の歴史において最大の転換点。C++98 から 13 年の歳月を経て、
ムーブセマンティクス、ラムダ式、スマートポインタなど、
現代的なプログラミングパラダイムが一気に導入されました。
「モダン C++」という呼称は C++11 以降を指します。
ここを理解していないと現代 C++ のコードは読めません。

### 2. C++20 — テンプレートと並行処理の進化

Concepts によりテンプレートのエラーメッセージが劇的に改善され、
Ranges により要素操作がパイプライン形式で書けるようになりました。
Coroutines と `std::jthread` により、
並行・非同期プログラミングの表現力が大幅に向上しました。

ただし Modules は普及が遅く、
Coroutines も使いどころが限定されているのが現状です。

### 3. C++23 — C++20 の現実化

`std::expected` により例外を使わないエラーハンドリングが標準化され、
`std::print` により `printf`/`iostream` に代わる型安全な出力が導入されました。
C++20 の革新的な機能を現場で使いやすくする「地固め」のリリースです。

### 4. 標準ライブラリの拡充

各バージョンで標準ライブラリが着実に拡充されており、
Boost に依存していた機能が次々と標準化されています
（`filesystem`、`optional`、`variant`、`format`、`flat_map` など）。

## バージョン間の関係

```text
C++98 → C++03 → C++11 → C++14 → C++17 → C++20 → C++23 → C++26(策定中)
                  ↑                        ↑
              言語リセット           第二の革命
```

- **C++98/03**: 古典的 C++（「C with Classes」の延長）
- **C++11/14**: モダン C++（ムーブ、ラムダ、スマートポインタ）
- **C++17/20**: ポストモダン C++（Concepts、Ranges、Coroutines）
- **C++23/26**: コンポーザブル C++（`expected`、モナド操作、リフレクション）

## 実務で押さえるべきライン

| レベル | 対象バージョン | 理由 |
| :--- | :--- | :--- |
| 最低ライン | **C++17** | 構造化束縛・`if constexpr`・`string_view` で実務が回る |
| 推奨 | **C++20** | Concepts + Ranges で設計品質が一段上がる |
| 最新志向 | **C++23** | `expected`・`print` で表現力と安全性がさらに向上 |
