# Go言語 バージョン別主要機能・進化の歩み

本ドキュメントは、Go 言語（Golang）のリリース 1.0 から最新の 1.26 までの歴史における、主要な機能追加および改善点をまとめたものです。

## 概要

Go は「後方互換性の維持」を最優先しつつ、6 ヶ月ごとのリリースサイクルで着実に進化を続けています。言語仕様の拡張だけでなく、標準ライブラリの拡充やランタイム（GC・スケジューラ）の最適化に多くのリソースを割いている点が特徴です。

## 歴史的転換点

### 1. 言語仕様の進化

* **Generics (1.18):** Go の歴史における最大の変更。型パラメータを使用可能にし、コードの再利用性を劇的に向上させました。
* **ループ変数スコープ (1.22):** 長年開発者を悩ませてきた「ループ変数の全イテレーション共有」問題がついに解消され、安全性が飛躍的に高まりました。

### 2. ランタイムとパフォーマンス

* **goroutineの最適化 (1.3/1.5/1.14):** 連続スタックへの変更、非同期プリエンプションの導入により、複雑な並行処理環境下でも安定したパフォーマンスを発揮できるようになりました。
* **GC (1.5/1.8):** 継続的な改善により、低遅延が求められるリアルタイムシステムでも Go が採用されるきっかけを作りました。

### 3. エコシステムとツールチェーン

* **Go Modules (1.11+):** 外部パッケージ管理を標準化し、Go 開発の現代的なワークフローを確立しました。
* **slog (1.21):** 構造化ログを標準ライブラリとして導入し、ログ出力の標準を統一しました。
